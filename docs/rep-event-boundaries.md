# REP Event Boundaries

The Reconstructible Event Protocol (REP) assumes that operational events
occur within identifiable temporal boundaries.

An Event Unit must define the interval during which the operational
action occurs.

## Temporal Boundaries

Each REP event includes two temporal markers:

time_start  
time_end  

These markers define the operational interval of the event.

## Event Emergence

An event begins when a decision initiates an operational action.

The start of the event corresponds to the moment the decision becomes
operationally effective.

## Event Closure

An event closes when its operational effects become observable and
evaluable.

The closure moment allows verification systems to evaluate the
reconstructibility of the event.

## Boundary Importance

Defining temporal boundaries prevents operational ambiguity.

Without identifiable event boundaries, the reconstruction of
Decision → Cost → Trace → Time becomes unreliable.

## Reconstruction

Temporal boundaries allow events to be placed within the operational
timeline and evaluated within the correct sequence.

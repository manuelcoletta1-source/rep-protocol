# REP Event Structure

The Reconstructible Event Protocol (REP) defines a minimal structure for
representing operational events.

An event is represented as an **Event Unit** containing the following fields:

event_id  
decision  
cost  
trace  
time_start  
time_end  

## event_id

A unique identifier assigned to the event.

The identifier allows events to be referenced and ordered within registries.

## decision

Description of the operational action that generated the event.

This field records the selection that initiated the operational change.

## cost

The localized loss associated with the decision.

Cost may represent:

- resource consumption
- economic expenditure
- computational load
- physical impact

The presence of cost indicates that the event introduced irreversible change.

## trace

Observable evidence generated by the event.

Trace records must remain reconstructible independently of the system that
originally recorded them.

Examples include:

- cryptographic hashes
- infrastructure state changes
- system logs
- external measurable effects

## time_start

Timestamp indicating when the event became operational.

## time_end

Timestamp indicating when the event becomes evaluable or closed.

## Reconstruction

The event structure enables deterministic reconstruction of the sequence:

Decision → Cost → Trace → Time

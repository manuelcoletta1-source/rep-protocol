# REP Glossary

This glossary summarizes the core concepts used in the Reconstructible Event Protocol (REP).

## Event

An operational occurrence that can be reconstructed through observable
evidence and temporal evaluation.

Within REP, an event is valid only when four conditions remain reconstructible:

Decision  
Cost  
Trace  
Time  

## Event Unit

The minimal structure used to represent an event within the REP model.

An Event Unit contains:

event_id  
decision  
cost  
trace  
time_start  
time_end  

## Decision

The selection that initiates an operational action.

A decision introduces irreversibility in the operational sequence.

## Cost

The localized loss associated with executing a decision.

Cost may appear as resource consumption, economic expenditure,
computational load, or physical impact.

## Trace

Observable evidence generated by the event.

Traces allow reconstruction of operational consequences independently
of the system that originally recorded them.

## Time

The interval during which the event remains observable and evaluable.

Temporal exposure allows verification of the persistence and evolution
of operational effects.

## Reconstructibility

The ability to deterministically reconstruct an operational event from
available evidence.

An event is reconstructible only when the four REP conditions remain
verifiable.

## Event Decay

The condition in which an event can no longer be reconstructed because
one or more required conditions are missing.

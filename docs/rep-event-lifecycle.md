# REP Event Lifecycle

The Reconstructible Event Protocol (REP) models the lifecycle of an operational event
from emergence to verification and eventual decay.

## Lifecycle Stages

An event passes through several stages during its operational existence.

### 1. Decision

A decision selects an operational action.

This step introduces irreversibility in the operational sequence.

### 2. Cost

The execution of the decision produces localized cost.

Cost may include resource expenditure, computational load,
economic impact, or physical effects.

### 3. Trace

The event generates observable traces.

These traces provide evidence that the operational action occurred.

### 4. Time Exposure

The event becomes exposed to time.

During this interval, the event can be evaluated, verified,
and reconstructed.

## Reconstruction

Verification reconstructs the event through the sequence:

Decision → Cost → Trace → Time

If all conditions remain observable, the event remains operationally valid.

## Decay

If one or more conditions disappear, the event becomes
non-reconstructible within the REP model.

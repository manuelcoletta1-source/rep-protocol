# Security Policy

This repository contains the specification of the Reconstructible Event Protocol (REP).

REP defines a minimal framework for reconstructible operational events.

## Reporting Issues

If you discover a security or protocol design issue, please report it through GitHub Issues.

Include:

- description of the issue
- potential impact
- suggested correction if available

## Scope

Security reports should focus on protocol-level concerns such as:

- event integrity
- trace verifiability
- registry tampering risks
- identity attribution mechanisms

Implementation-specific vulnerabilities should be reported to the corresponding implementation repository.

## Status

REP is currently an experimental protocol draft.

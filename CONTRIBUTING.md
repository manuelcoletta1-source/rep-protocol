# Contributing to REP

This repository contains the working draft of the Reconstructible Event Protocol (REP).

Contributions should focus on improving clarity, technical precision, and protocol coherence.

## Guidelines

- Keep the protocol minimal.
- Avoid adding unnecessary complexity.
- Preserve the core event model:

Decision  
Cost  
Trace  
Time

## Process

1. Fork the repository.
2. Create a branch for your changes.
3. Submit a pull request with a clear description of the modification.

All changes should improve the reconstructibility and operational clarity of the protocol.

## Scope

REP is a protocol specification. Implementation details should remain outside the core document unless necessary for protocol understanding.
